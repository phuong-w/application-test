Introduction

Base URL: http://localhost:8000

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Authentication Endpoints

APIs for authenticating user

Login.

This endpoint let the user login to the system and return them with a token for authenticate on authenticated endpoints

Example request:
curl --request POST \
    "http://localhost:8000/api/auth/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "API-KEY: VAND2023Staging" \
    --data "{
    \"email\": \"user@test.com\",
    \"password\": \"password\"
}"

const url = new URL(
    "http://localhost:8000/api/auth/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "API-KEY": "VAND2023Staging",
};

let body = {
    "email": "user@test.com",
    "password": "password"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


{
    "data": {
        "id": 32,
        "name": "Liam Lubowitz",
        "email": "felix33@example.org",
        "token": null
    }
}

Product Endpoints

Get a list of products.

requires authentication

This endpoint lets you get a list of products

Example request:
curl --request GET \
    --get "http://localhost:8000/api/v1/product?limit=50" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --header "API-KEY: VAND2023Staging"
const url = new URL(
    "http://localhost:8000/api/v1/product"
);

const params = {
    "limit": "50",
};
Object.keys(params)
